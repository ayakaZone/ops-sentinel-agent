# 第一组优化：文档清洗误删修复

## 1. 问题背景

原来的清洗逻辑会在整篇文档范围内统计重复文本，并删除重复内容。这种做法会误删 RAG 知识中的有效正文，例如：

- 多篇文档都出现的工具名 `query_logs`；
- Markdown 代码块中反复出现的调用示例；
- 为强调而在不同位置重复的处理建议。

RAG 场景中，**少量重复内容通常只会带来少量相似 Chunk；正文被误删则会让知识无法被检索到**。因此本次优化选择保守清洗策略。

## 2. 优化目标

1. 删除 PDF 中跨页重复的页眉、页脚和页码。
2. 保留正文、代码块和不相邻的重复段落。
3. 仅删除“完整内容相同且相邻”的重复段落。
4. 保证写入 Milvus 前不保留 PDF 的分页分隔符。

## 3. PDF 页眉页脚的两阶段算法

核心原则：**先确认，再删除。候选文本不等于一定删除。**

```text
PDF 加载并保留页边界
→ 按页拆分
→ 取每页顶部、底部候选行
→ 页码归一化
→ 跨页计数
→ 达到阈值后确认重复页眉/页脚
→ 仅在页面边缘删除已确认文本
→ 拼回完整文本
```

### 3.1 保留 PDF 页面边界

加载器通过 `pdfplumber` 遍历 PDF 页面，并用 `\f`（换页符）拼接页面文本：

```python
return PDF_PAGE_SEPARATOR.join(page_texts)
```

`\f` 只在“加载到清洗”这个过程内使用，用于判断某行是否位于某页顶部或底部。清洗完成后，页面会用普通换行重新拼接，因此不会把 `\f` 写入向量库。

### 3.2 候选行不是已确认的页眉页脚

默认每页取前两行、后两行作为候选。例如一页有 10 行：

```python
header_indexes = {0, 1}
footer_indexes = {8, 9}
```

这只是因为页眉、页脚通常出现在页面边缘，且可能各占两行；候选内容还需要跨页统计，不能立即删除。

### 3.3 页码归一化

不同页面的页码文字不同：

```text
第 1 页
第 2 页
第 3 页
```

通过正则替换为同一模板：

```text
第 {page} 页
```

当前实现使用 `pattern.sub(替换文本, 原文本)`：它会替换一行中符合正则规则的部分，因此也能处理 `运维手册 - 第 3 页` 这种含有固定文字的页脚。

### 3.4 跨页计数与阈值

`Counter` 的含义是“候选文本出现于多少个页面”。每页候选内容先放入 `set`，保证同一页即使重复两次也只计一次。

```python
header_counts.update(current_page_headers)
footer_counts.update(current_page_footers)
```

默认阈值为 `ceil(总页数 * 0.6)`，同时至少要求两页。例如 3 页 PDF 至少在 2 页出现，才会被认定为重复页眉或页脚。

### 3.5 第二遍精准删除

第一遍只能得到：

```python
repeated_headers = {"运维知识库"}
repeated_footers = {"第 {page} 页"}
```

第二遍遍历每一页的行。某行只有同时满足以下条件才会删除：

1. 它处于当前页的顶部或底部候选位置；
2. 它归一化后属于已确认的 `repeated_headers` 或 `repeated_footers`。

这样正文中即使出现相同的工具名或句子，也不会因为“重复”被删除。

## 4. 相邻段落去重

页眉页脚清洗完成后，代码只去除“完整相同且相邻”的段落：

```text
段落 A
段落 A
→ 保留一个段落 A
```

下列情况均保留：

- 两个段落仅有某一句相同；
- 两个完全相同的段落不相邻；
- Markdown 代码块中的重复工具调用。

这是一个保守策略，优先保障知识完整性。

## 5. 相关代码位置

| 文件 | 作用 |
| --- | --- |
| `app/services/document_loader_service.py` | PDF 加载，并保留 `\f` 页面边界。 |
| `app/services/document_cleaner_service.py` | 页感知页眉页脚清洗、页码归一化、相邻段落去重。 |
| `app/services/vector_index_service.py` | 索引前将文件扩展名传给清洗服务，使仅 PDF 走分页清洗。 |
| `tests/services/test_document_cleaner_service.py` | 覆盖页眉页脚、页码、正文重复、代码块与相邻段落的回归测试。 |

## 6. 验证结果

- 文档清洗单元测试：12 项通过。
- 非集成测试：19 项通过，9 项按标记跳过。
- 项目 Markdown 文档代码围栏数量：清洗前后均为 32。
- 清空并重建 Milvus `biz` 集合后，20 份知识文档全部索引成功。

## 7. 面试表达

> 我发现全局按重复文本去重会误删 Markdown 中的工具调用和代码内容，因此将 PDF 清洗改为页感知的两阶段策略：第一阶段仅统计每页顶部、底部候选文本，归一化页码后按照跨页出现比例识别页眉页脚；第二阶段只删除已确认文本在页面边缘的位置。段落去重限制为完整且相邻的重复段落，优先避免 RAG 知识被误删。
